下面按你提到的 5 个场景做“简单用户旅程地图”，并在每个场景后列出它反向要求的能力。最后再给一份跨场景的能力清单，便于你做重构优先级。

**电商独立站（移动端）**  
旅程地图（买家）
- 进入首页 → 浏览推荐/分类  
- 搜索或筛选 → 进入商品详情  
- 选择规格/数量 → 加入购物车  
- 结算（地址/支付方式） → 下单成功  
- 订单详情 → 查看物流/售后

旅程地图（商家/运营）
- 创建店铺 → 上传商品/价格/库存  
- 配置首页/促销 → 发布  
- 订单列表 → 发货/更新状态  
- 数据 tab → 查看注册、下单、转化

反向要求的能力
- 数据模型：Product/Category/Cart/Order/User  
- 表单与列表：商品录入、订单管理、用户管理  
- 交易流：购物车、结算、订单状态机（可先 mock）  
- 事件埋点：注册、加购、下单、支付成功  
- 生成能力：商品卡片/详情页/购物车/结算页/订单页模板  
- 资产能力：品牌 logo + 商品图 + 风格图驱动视觉

---

**旅行行程整合与分发**  
旅程地图（组织者）
- 新建行程 → 输入订单信息（航班/酒店/交通/餐厅/景点）  
- 归类到日期/时间段 → 自动生成行程安排  
- 生成分享页 → 发送给朋友  
- 朋友查看 → 反馈或补充信息

旅程地图（参与者）
- 打开分享页 → 查看总览  
- 按天查看 → 获取具体细节  
- 标记收藏或截图

反向要求的能力
- 数据模型：Trip/Booking/DayPlan/Location/Contact  
- 结构化输入：多表单 + 批量导入（手动）  
- 视图模板：日程列表/时间轴/卡片详情  
- 分享能力：只读链接、权限（可选）  
- 资产能力：背景图作为封面，logo 作为行程标识

---

**说明书网站（多页）**  
旅程地图（作者）
- 创建产品 → 创建章节/页面  
- 填写内容 + 插图 → 预览  
- 发布 → 生成链接

旅程地图（读者）
- 进入目录 → 选择章节  
- 阅读单页 → 上一页/下一页  
- 搜索关键词（可选）

反向要求的能力
- 数据模型：Manual/Section/Page  
- 内容编辑：富文本或 Markdown  
- 导航组件：目录、分页、面包屑  
- 多页路由：静态多页或 SPA 动态  
- 资产能力：插图/示意图 + 风格统一

---

**任务管理看板**  
旅程地图（团队成员）
- 创建看板 → 设置列（To Do / Doing / Done）  
- 创建任务 → 分配/标签/优先级  
- 拖动任务 → 更新状态  
- 查看任务详情 → 评论/附件（可选）

反向要求的能力
- 数据模型：Board/Column/Task/User/Tag  
- UI 组件：列、卡片、详情面板  
- 交互：拖拽（可先静态）  
- 权限：协作/只读  
- 数据展示：统计（完成率、逾期）

---

**Landing Page（移动端）**  
旅程地图（访客）
- 首屏抓住注意力 → 了解价值  
- 浏览功能/案例 → 信任增强  
- 查看价格/FAQ → 转化  
- 提交表单/跳转下载

反向要求的能力
- 组件模板：Hero/Features/Testimonials/Pricing/FAQ/CTA  
- CTA 表单：邮箱/手机号收集  
- 资产能力：背景图 + 风格图  
- 埋点：点击/转化

---

**跨场景的核心新增能力（按优先级）**

1) 结构化输入与建模  
- 场景识别 → 输出数据模型 + 页面清单  
- 表单生成（Schema 驱动）  
- 多图多标签资产注册与用途绑定

2) 页面/组件模板系统  
- 页面级模板（电商/行程/说明书/看板/landing）  
- 组件级模板（卡片、列表、详情、表单、时间轴）

3) 数据流与状态  
- 统一数据实体 + demo 数据生成  
- 事件/埋点规范（注册、下单、保存、分享）  
- 简易数据看板（表格 + 统计卡）

4) 视觉与品牌  
- 风格图 → 颜色/排版/组件样式约束  
- 背景图 → 首屏/分区背景  
- Logo → 顶栏/页脚/分享页

5) 发布与分享  
- 可预览版本  
- 只读分享链接  
- 导出（可选：PDF/静态站）
